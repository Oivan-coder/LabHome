# Health ID — MVP личного кабинета пациента

Статический прототип без БД, авторизации и серверной части.

## Что внутри

- `index.html` — структура приложения.
- `styles.css` — Apple-like визуальный стиль, адаптивная верстка.
- `app.js` — моковые данные пациента, приемы, заключения, лабораторные показатели и графики на canvas.

## Как запустить

Откройте `index.html` в браузере.

Логин демо:
- телефон: любой
- номер карты МИС: любой

## Что можно показать инженерам

Сейчас данные зашиты в `mockData` внутри `app.js`. В боевой версии этот объект заменяется вызовами API:

```http
GET /patients/{mis_card}/summary
GET /patients/{mis_card}/visits
GET /patients/{mis_card}/reports
GET /patients/{mis_card}/lab-results
```

Для лабораторного блока желательно перейти на FHIR-структуру:
- `DiagnosticReport` — отчет/исследование;
- `Observation` — отдельный показатель;
- справочник локальных кодов ЛИС/МИС → LOINC/нормализованный код.

## Ограничения MVP

Это визуальный и продуктовый прототип. Здесь нет:
- настоящей авторизации;
- проверки прав пациента;
- защиты персональных данных;
- серверной части;
- подключения к МИС/ЛИС;
- медицинской валидации интерпретаций.
